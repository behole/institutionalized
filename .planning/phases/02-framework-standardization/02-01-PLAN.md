**Agent files (prosecutor.ts, defense.ts, jury.ts, judge.ts):**
1. Remove `import Anthropic from "@anthropic-ai/sdk"` from all 4 files
2. Remove module-level `const anthropic = new Anthropic(...)` singleton from all 4 files
3. Add `import type { LLMProvider } from "@core/types"` to all 4 files
4. Add `import { parseJSON } from "@core/orchestrator"` where JSON parsing is needed
5. Add `provider: LLMProvider` parameter to each agent function signature (prosecute, defend, deliberate, renderVerdict)
6. Replace `anthropic.messages.create({...})` calls with `provider.call({...})` -- map `max_tokens` to `maxTokens`, `messages` stays the same structure
7. Replace manual JSON parsing (`content.text.trim()`, `text.match(/\{[\s\S]*\}/)`, `JSON.parse`) with `parseJSON<T>(response.content)`
8. Keep `validateExhibits` and all pure validation logic unchanged (no LLM calls in those)
9. For jury.ts: keep juror parallelism pattern but accept provider param -- the orchestrator will handle FrameworkRunner parallelism

**orchestrator.ts:**
1. Add `import { FrameworkRunner } from "@core/orchestrator"` and `import type { LLMProvider } from "@core/types"`
2. Add `provider: LLMProvider` parameter to `runCourtroom()` signature
3. Create `const runner = new FrameworkRunner<Case, CourtroomResult>("courtroom", caseInput)`
4. Replace direct agent calls with runner-wrapped calls:
   - `prosecute(caseInput, config)` becomes `prosecute(caseInput, config, provider)` -- agent still calls provider.call() directly, but orchestrator ALSO wraps key phases with runner.runAgent() for cost tracking. OR: refactor agent functions to return prompt strings and have orchestrator call runner.runAgent() then pass response to agent parse functions.

   RECOMMENDED APPROACH (simpler, per research Pattern 4):
   - Agent functions keep their current structure but gain `provider` param and use `provider.call()` instead of `anthropic.messages.create()`
   - In orchestrator, wrap each agent call: call agent function, then manually record the cost step in the audit trail. Actually -- this won't work because runner.runAgent() IS the choke-point for cost tracking.

   CORRECT APPROACH:
   - For sequential agents (prosecutor, defense, judge): call `runner.runAgent()` in orchestrator.ts with the prompt, then parse the response using the agent's parse logic. Alternatively, have the agent function call provider.call() directly and have the orchestrator record the step manually via `runner.getAuditTrail().recordStep()`.

   SIMPLEST CORRECT APPROACH (use this):
   - Agent files export TWO things: (1) a prompt-building function that returns {systemPrompt, userPrompt}, and (2) a response-parsing function that takes string and returns the typed result.
   - orchestrator.ts uses `runner.runAgent(name, provider, model, userPrompt, temp, maxTokens, systemPrompt)` then calls the parse function on `response.content`.
   - This keeps all prompt logic in agent files, all orchestration in orchestrator.ts, and cost tracking automatic via FrameworkRunner.

   For jury specifically:
   - jury.ts exports `buildJurorPrompt(caseInput, prosecution, defense, config, jurorIndex)` and `parseJurorVerdict(text)`
   - orchestrator.ts uses `runner.runParallel()` with config.parameters.jurySize jurors, then parses each response

5. Remove `const costUSD = 0.0; // Placeholder`
6. After all phases, call `const { auditLog } = await runner.finalize(result, "complete")`
7. Set `metadata.costUSD = auditLog.metadata.totalCost`
8. Return the same CourtroomResult shape -- verify against the fixture from Task 1
