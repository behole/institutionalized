**For each framework, the migration has 3 layers:**

**Layer 1 -- Agent files (dialectic.ts, peer.ts, facilitator.ts, interpreters.ts):**
1. Remove `import { generateObject } from "@institutional-reasoning/core"`
2. Add `import { parseJSON } from "@core/orchestrator"` and `import type { LLMProvider } from "@core/types"`
3. Add `provider: LLMProvider` parameter to each agent function
4. Replace `generateObject<T>({model, system, prompt, temperature})` with:
   ```typescript
   const response = await provider.call({
     model,
     messages: [{ role: "user", content: prompt }],
     temperature,
     systemPrompt: system,
     maxTokens: 4096,
   });
   return parseJSON<T>(response.content);
   ```
5. Keep the try/catch fallback pattern that exists in each function -- just update the inner call

**Layer 2 -- Orchestrator files (orchestrator.ts):**
1. Add `import { FrameworkRunner } from "@core/orchestrator"` and `import type { LLMProvider } from "@core/types"`
2. Add `provider: LLMProvider` parameter to the main orchestrator function (runDialectic, runWorkshop, runTalmudic)
3. Create `const runner = new FrameworkRunner<InputType, ResultType>(frameworkName, input)`
4. Pass `provider` through to each agent function call
5. Remove `const costUSD = 0.0; // Placeholder`
6. After all agent calls: `const { auditLog } = await runner.finalize(result, "complete")`
7. Set `metadata.costUSD = auditLog.metadata.totalCost`

IMPORTANT for cost tracking: Since agent functions call `provider.call()` directly (not via `runner.runAgent()`), the orchestrator must manually record cost steps. Two options:
- Option A: Have orchestrator call `runner.runAgent()` and pass response to agent parse function (requires splitting agent functions into prompt-builder + parser)
- Option B: After each agent call, manually record via `runner.getAuditTrail().recordStep(agentName, model, prompt, response, duration, provider.calculateCost(response.usage, model))`

Use Option A (consistent with courtroom pattern from Plan 01). Refactor agent functions to export prompt builders and response parsers, then orchestrator uses `runner.runAgent()`.

**CRITICAL for writers-workshop:** Current code runs peer reviews sequentially via `for` loop. PRESERVE this sequential behavior -- use `runner.runAgent()` in a loop, NOT `runner.runParallel()`. The structure IS the value.

**Layer 3 -- index.ts:**
1. Add `import { createProvider } from "@core/providers"` and `import { getAPIKey } from "@core/config"`
2. In run(), create provider: `const provider = createProvider({ name: flags.provider || "anthropic", apiKey: getAPIKey(flags.provider || "anthropic") })`
3. Pass provider to orchestrator function
4. run() signature stays IDENTICAL
